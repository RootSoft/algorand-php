Get a specific pending transaction.

public getPendingTransactionById(string $transactionId) : PendingTransaction

Given a transaction id of a recently submitted transaction, it returns information about it. There are several cases when this might succeed:

  • transaction committed (committed round > 0)
  • transaction still in the pool (committed round = 0, pool error = "") (committed round = 0, pool error != "")

Or the transaction may have happened sufficiently long ago that the node no longer remembers it, and this will return an error.

waitForConfirmation()

ManagesTransactionsV2.php : 174

Utility function to wait on a transaction to be confirmed.

public waitForConfirmation(string $transactionId[, int $timeout = 5 ]) : PendingTransaction

The timeout parameter indicates how many rounds do you wish to check pending transactions for.

On Algorand, transactions are final as soon as they are incorporated into a block and blocks are produced, on average, every 5 seconds.

This means that transactions are confirmed, on average, in 5 seconds
