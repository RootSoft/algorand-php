Call a Stateful Smart Contract.

public call(Account $account, BigInteger $applicationId, array<string|int, mixed> $arguments[, bool $waitForConfirmation = false ][, int $timeout = 5 ]) : PendingTransaction|string

This is a helper function to easily call a stateful smart contract and pass arguments to it.

Arguments can be passed to any of the supported application transaction calls, including create. The number and type can also be different for any subsequent calls to the stateful smart contract.

Once an account has opted into a stateful smart contract it can begin to make calls to the contract. These calls will be in the form of ApplicationCall transactions that can be submitted with goal or the SDK.

Depending on the individual type of transaction as described in The Lifecycle of a Stateful Smart Contract, either the ApprovalProgram or the ClearStateProgram will be called.

Generally, individual calls will supply application arguments.

deleteApplication()

ManagesApplicationsV2.php : 350

Deletes an existing Stateful Smart Contract.

public deleteApplication(Account $account, BigInteger $applicationId[, bool $waitForConfirmation = false ][, int $timeout = 5 ]) : PendingTransaction|string

This is a helper function to easily delete a stateful smart contract.

To delete a smart contract, an ApplicationCall transaction of type DeleteApplication must be submitted to the blockchain. The ApprovalProgram handles this transaction type and if the call returns true the application will be deleted. The approval program defines the creator as the only account able to delete the application. This removes the global state, but does not impact any user's local state.

optIn()

ManagesApplicationsV2.php : 153

Opt in to the smart contract.

public optIn(Account $account, BigInteger $applicationId[, bool $waitForConfirmation = false ][, int $timeout = 5 ]) : PendingTransaction|string

This is a helper function to opt in to an existing smart contract.

Before any account, including the creator of the smart contract, can begin to make Application Transaction calls that use local state, it must first opt into the smart contract.

This prevents accounts from being spammed with smart contracts. To opt in, an ApplicationCall transaction of type OptIn needs to be signed and submitted by the account desiring to opt into the smart contract.

update()

Updates an existing Stateful Smart Contract.

public update(Account $account, BigInteger $applicationId, TEALProgram $approvalProgram, TEALProgram $clearStateProgram[, array<string|int, mixed>|null $arguments = null ][, bool $waitForConfirmation = false ][, int $timeout = 5 ]) : PendingTransaction|string

This is a helper function to easily update a stateful smart contract.

A stateful smart contract’s programs can be updated at any time. This is done by an ApplicationCall transaction type of UpdateApplication. This operation can be done with goal or the SDKs and requires passing the new programs and specifying the application ID.

The one caveat to this operation is that global or local state requirements for the smart contract can never be updated.
